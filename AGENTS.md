# Codex Project Instructions

## Project Nature

本项目是面向专业命理师傅的八字结构化研判辅助网站，目标是减轻重复分析压力，帮助师傅快速看到断事主线、证据链与复核条件。

后续 Codex 修改时，应围绕“专业研判、证据可追溯、师傅可复核”的产品定位展开。系统可以输出断事倾向，但必须说明断法依据、现实应象、成立条件和反证条件，避免把单一象义包装成无证据的结论。

## Page Narrative

页面主线必须遵守以下顺序：

1. 排盘结果
2. 一句话总览
3. 证据链
4. 专业复核卡片
5. 大运流年

## Interpretation Rules

不要新增确定性断语。

禁止输出以下词语或表达：

- 一定
- 必定
- 绝对
- 必然
- 必离婚
- 必发财
- 必有灾
- 必坐牢
- 必死亡

所有解读必须使用专业复核型表达，例如：

- 主断倾向
- 断法依据
- 现实应象
- 成立条件
- 反证条件
- 师傅复核点

## Development Constraints

修改代码后必须运行：

```bash
node scripts/validate-bazi-data.mjs
npm test
```

不要删除已有模块。

不要大规模重构。

本周目标是可用 Beta，不追求内容很多，优先保证页面清晰、稳定、不报错。
